# 提取产物输出规范

本文档定义 extract-workflow 所有提取产物的输出路径、文件格式与内容要求。

## 📁 输出目录

| 产物类型 | 输出路径 | 说明 |
|---------|---------|------|
| 视觉规范 | `src/themes/visual-spec.md` | 项目统一视觉规范，被所有原型引用 |
| 示例数据 | `src/data/<name>.md` | 每个数据对象一个文件，原型创建时按需复用 |

## 📄 视觉规范格式（`src/themes/visual-spec.md`）

## ✅ 写入规则

- **覆盖策略**：`src/themes/visual-spec.md` 已存在时，提示用户确认是否覆盖或合并，不自动覆盖
- **新增策略**：`src/data/` 中已有同名文件时，默认追加或扩展，不覆盖已有字段
- **来源标注**：每个文件头部注明提取来源与时间，便于追溯
- **不确定项**：无法确认的色值、字段含义，用 `待确认` 标注，不凭空填写
