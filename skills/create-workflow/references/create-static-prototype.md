# 创建静态原型

创建一个完整页面平铺展示的静态 HTML 原型，适用于设计走查、内容评审与布局确认。

## 角色定位

你将作为 **产品经理 × 交互设计师（复合型）**，协助用户完成静态原型的创建。产出物为所有页面状态平铺展示的 HTML 文件，面向设计走查与内容评审，不实现任何交互逻辑。

## 核心流程

阅读规范 → 需求对齐 → 设计与 spec.md → 产出静态 HTML

### 步骤 1：阅读规范文档

**必须完整阅读以下文档**，不可跳过：

1. `../rules/design-guide.md` —— 设计流程总纲（资料收集、内容规划、视觉设计、规格产出）
2. `../rules/visual-spec.md` —— 视觉规范的查找与应用步骤
3. `../templates/spec-template.md` —— 规格文档模板

### 步骤 2：需求对齐

判断用户提供的信息是否足够开始设计，按以下情况处理：

- **已有 `spec.md`**：直接跳到步骤 3，不需要重新对齐
- **需求已明确**（用户描述了页面目标、功能点、使用场景）：直接进入步骤 3
- **需求不足**：使用以下模板补齐，等待用户回复后再继续

```text
收到，准备创建静态原型。

请详细描述您的需求：
```

### 步骤 3：设计与规格文档

**无 `spec.md` 时必须完整执行并写入；已有 `spec.md` 时按需执行 3.1–3.3 完善数据与视觉，3.4 更新已有文件。**

按以下顺序执行：

#### 3.1 前置检索

- 按 `../rules/design-guide.md` 的"资料收集"完成项目资产检索（`src/data/`、`src/prototypes/`）

#### 3.2 维护示例数据

检索 `src/data/` 后按以下情况处理：

- **复用**：已有文件且字段完全够用 → spec.md 中引用 `src/data/<name>.md`，不改动数据文件
- **扩展**：已有文件但需新增字段 / 补充示例 → 直接编辑 `src/data/<name>.md`，在原有结构上追加，不删除已有字段；spec.md 引用该文件
- **新建**：无匹配文件且数据量较小 → 新建 `src/data/<name>.md`（格式见下方），spec.md 引用该文件
- **批量提取场景**：如需从设计稿一次性提取大量数据对象，引导用户走 `extract-workflow`，不在本流程中处理

`src/data/<name>.md` 简洁格式：

```markdown
# [数据名称]

> 用途说明：供哪些原型页面使用

## 字段定义

| 字段名 | 说明 | 类型 | 示例值 |
|--------|------|------|--------|
| field1 | 说明 | 文本 | 示例   |

## 示例数据

| field1 | field2 |
|--------|--------|
| 示例1  | 示例1  |
```

扩展已有文件时保留原"用途说明"与历史字段，仅在表格末尾追加新行；如有变动，在文件头补一行 `> 最近更新：YYYY-MM-DD（新增/调整说明）`。

#### 3.3 确定视觉与布局

- 按 `../rules/design-guide.md` 的"视觉设计流程"确定视觉规范与布局方向
- 在 spec.md 的"信息架构"中列出需要平铺的所有页面状态与弹窗，作为步骤 4 的平铺清单

#### 3.4 写入 spec.md

- 使用 `../templates/spec-template.md` 模板生成并**写入** `spec.md`
- 用以下格式向用户汇报变更，再继续步骤 4：

```text
✅ spec.md 已写入：src/prototypes/<page-name>/spec.md

数据文件变更：
- [复用 / 扩展 / 新建] src/data/<name>.md

接下来开始产出静态 HTML 原型。
```

### 步骤 4：产出静态 HTML 原型

#### 4.1 评估规模，决定实现策略

根据 spec.md 中需要平铺的页面状态与弹窗数量判断规模：

- **小型**（平铺区块 ≤ 4 个）：直接一次性实现，进入 4.3
- **大型**（平铺区块 > 4 个）：进入 4.2 拆分实现

#### 4.2 拆分实现计划（仅大型）

按页面或模块拆分实现计划，使用 TodoWrite 登记每个区块为独立任务：

- 每个任务对应一个完整页面或一组相关状态（如：列表页默认态 + 空态、详情页、新建弹窗等）
- 实现顺序：**整体骨架与导航 → 主页面 → 子页面 → 弹窗与浮层**
- 每完成一个区块立即标记为 completed，再启动下一个
- 所有区块共享样式，统一收敛到文件顶部

**文件写入策略（必须遵守）**：
- **第一个 task**：用 `Write` 创建 `index.html`，写入 `<html>`、`<head>`、CSS 变量、全局样式、骨架结构，以 `</body></html>` 结尾
- **后续每个 task**：用 `Edit` 在 `</body>` 前插入当前区块的 HTML 内容，不重写整个文件
- 每个 task 只写自己负责的区块，严禁重新生成已有内容

将拆分计划告知用户后再开始实现。

#### 4.3 产出要点

**静态平铺规则**：

- 不编写任何 JavaScript 交互逻辑（无点击、切换、表单提交等）
- 所有页面状态、视图与流程步骤作为独立区块，按顺序垂直排列在同一 HTML 文件中
- 每个区块用标题标注说明，格式：`<!-- 页面：XXX -->` 注释 + 可见的区块标题标签
- **弹窗 / 抽屉 / 下拉浮层**：不触发展开，直接在页面流中追加显示，在区块标题处注明"弹窗：XXX"
- **多步骤流程**（向导、分步表单）：每步作为独立区块顺序平铺，标注"步骤 N：XXX"
- 区块之间用明显的分隔线与背景色区分，便于走查时快速定位

**实现规范**：

- 用 `Write` 创建 `index.html`，一次性写入完整内容（小型原型）；大型原型按 4.2 策略用 `Edit` 追加
- 所有样式内联或写在同一文件内，确保单文件可独立打开预览
- 使用原生 HTML + CSS 实现页面结构与样式
- 严格遵循 `src/themes/visual-spec.md` 中的视觉规范

## 输出文件

- `src/prototypes/<page-name>/spec.md` - 规格文档
- `src/prototypes/<page-name>/index.html` - 静态原型
